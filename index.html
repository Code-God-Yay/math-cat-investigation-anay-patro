Introduction to recursion
What is recursion?

A function that calls itself to solve a smaller version of the same problem. It breaks big problems into identical smaller steps, each one simpler than the last — until it hits a stopping point called the base case.

Why use it for fractals?

Fractals are shapes that look the same at every scale — a perfect match for recursion. A Level 5 tree contains two Level 4 trees, which each contain two Level 3 trees, and so on. One function, any depth.

Real-world link

Trees, snowflakes, coastlines, ferns, river deltas, and even DNA replication all follow recursive, self-similar patterns. Recursion is nature's favorite design principle.

The two rules

Base case: the condition that stops recursion — without it, the function would crash with a stack overflow.

Recursive case: the call to itself with a simpler input (n−1), guaranteeing we approach the base case.

Task 6
The Fibonacci sequence
Python — Fibonacci
def fibonacci(n): if n <= 2: return 1 # base cases: F(1)=F(2)=1 return fibonacci(n-1) + fibonacci(n-2) # fibonacci(10) → 55
F(n) = F(n−1) + F(n−2)   where F(1) = F(2) = 1
Two base cases

Unlike previous tasks, Fibonacci needs two base cases: F(1) = 1 and F(2) = 1. This is because each step depends on the two steps before it, not just one — so we need two anchors to stop the recursion.

Found in nature

The golden ratio

As n grows, F(n)/F(n−1) approaches φ ≈ 1.618 — the golden ratio. This is why Fibonacci patterns look proportionally pleasing in nature and architecture.

Task 10
Introducing randomness
Python — randomfractaltree
import random def randomfractaltree(n, length, angle, scale): if n > 0: pen.forward(length) pos = pen.position() head = pen.heading() # angle jitter: ±15° jitter = random.randint(-15, 15) # scale jitter: ±10% sj = scale * random.uniform(0.9, 1.1) pen.left(angle + jitter) randomfractaltree(n-1, length*sj, angle, scale) pen.goto(pos); pen.setheading(head) pen.right(angle + jitter) randomfractaltree(n-1, length*sj, angle, scale) pen.goto(pos); pen.setheading(head)
What's random — and why

Two sources of variation are added per branch:

A — Angle jitter (±15°)

Mimics wind, light-seeking, and uneven growth. Real branches don't grow at exact symmetric angles.

B — Scale jitter (±10%)

Variation in branch length reflects genetic and environmental noise. No two real branches grow identically.

Randomness mimics nature's noise:
wind · sunlight · genetics · soil
Random tree — different every run
Result

Each call to randomfractaltree(10, 80, 30, 0.8) produces a different tree every time — just like real trees grown from the same species in different locations.

The big idea

Complex shapes found in nature — trees, coastlines, snowflakes — can be described by very simple recursive rules. One function with a base case and a recursive call is all it takes to grow an infinitely detailed fractal. Nature is recursive at its core.

Technical challenge

Fixing the _tkinter library error on Mac required configuring the correct Python framework build — a real-world reminder that environment setup is a core part of programming, not an obstacle.

Math encountered

Recurrence relations, geometric scaling (powers of the scale factor), triangular numbers, and the connection between Fibonacci numbers and natural growth all surfaced naturally through the code.

Design insight

Adding randomness was the most revealing step — tiny noise made synthetic trees look genuinely organic. Real complexity often emerges from simple rules plus random variation.

Git & version control

Maintaining v1.3 as a stable release while experimenting built good software development habits — commit early, tag releases, branch for experiments.
