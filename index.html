Concept
Introduction to recursion
What is recursion?

A function that calls itself to solve a smaller version of the same problem. It breaks big problems into identical smaller steps, each one simpler than the last.

Why use it for fractals?

Fractals are shapes that look the same at every scale — the perfect match for recursion. A Level 5 tree contains two Level 4 trees, which each contain two Level 3 trees, and so on. One function, any depth.

Real-world link

Trees, snowflakes, coastlines, ferns, river deltas, and even DNA replication all follow recursive, self-similar patterns found in nature.

The simplest recursive function
def printdots(n): if n > 0: print(".") printdots(n - 1) # calls itself!
Two rules every recursive function needs

Base case: the condition that stops the recursion (e.g. if n > 0) — without it, the function crashes.

Recursive case: the call to itself with a simpler input (n - 1).

Tasks 4 & 5
The math logic — stars & triangular numbers
Task 4 — star triangle
def startriangle(n): if n > 0: print("*" * n) startriangle(n - 1) # startriangle(5) outputs: # ***** # **** # *** # ** # *
Key insight

Each call prints n stars then calls itself with n−1. The base case (n = 0) stops the chain. User input was added so anyone can test any number.

Task 5 — triangular number
def triangular(n): if n <= 0: return 0 return n + triangular(n - 1) # triangular(6) → 21 # because 6+5+4+3+2+1 = 21
T(n) = n + T(n−1)   with T(0) = 0
The recursive relationship

The n-th triangular number equals n plus the (n−1)-th triangular number. This recurrence relation maps perfectly to a recursive function — each call delegates the smaller sum to itself.

Task 6
The Fibonacci sequence
Recursive implementation
def fibonacci(n): if n <= 2: return 1 # base case return fibonacci(n-1) + fibonacci(n-2) # fibonacci(10) → 55
F(n) = F(n−1) + F(n−2)  where F(1) = F(2) = 1
Two base cases

Unlike previous tasks, Fibonacci needs two base cases: F(1) = 1 and F(2) = 1. This is because each step depends on the two steps before it, not just one.

Tasks 7 & 8
Growing the first fractal tree
Task 7 — simpletree(n)
def simpletree(n): if n > 0: pen.forward(50) pos = pen.position() head = pen.heading() pen.left(30) simpletree(n - 1) # left branch pen.goto(pos); pen.setheading(head) pen.right(30) simpletree(n - 1) # right branch pen.goto(pos); pen.setheading(head)
Why this works

The function saves its position and heading after each trunk segment, then recursively draws a left sub-tree, returns, and draws the right sub-tree. This directly mirrors the hand-drawn instructions from Part 1.

Task 8 — fractaltree(n, length, angle, scale)
def fractaltree(n, length, angle, scale): if n > 0: pen.forward(length) pos = pen.position() head = pen.heading() pen.left(angle) fractaltree(n-1, length*scale, angle, scale) pen.goto(pos); pen.setheading(head) pen.right(angle) fractaltree(n-1, length*scale, angle, scale) pen.goto(pos); pen.setheading(head)
n = level length = trunk px angle = branch ° scale = shrink ratio
Test: fractaltree(8, 100, 30, 0.75)

At scale 0.75, each level's branches are 75% the length of the previous — creating natural tapering toward the tips.

Task 10
Introducing randomness
randomfractaltree(n, length, angle, scale)
import random def randomfractaltree(n, length, angle, scale): if n > 0: pen.forward(length) pos = pen.position() head = pen.heading() # jitter angle by up to ±15° jitter = random.randint(-15, 15) # jitter scale by ±10% s_jitter = scale * random.uniform(0.9, 1.1) pen.left(angle + jitter) randomfractaltree(n-1, length*s_jitter, angle, scale) pen.goto(pos); pen.setheading(head) pen.right(angle + jitter) randomfractaltree(n-1, length*s_jitter, angle, scale) pen.goto(pos); pen.setheading(head)
What's random — and why

Two sources of variation are added per branch:

  • A
    Angle jitter: ±15° from the base angle mimics wind, light-seeking, and uneven growth.
  • B
    Scale jitter: ±10% variation in branch length reflects genetic and environmental noise.
Result

Calling randomfractaltree(10,80,30,0.8) produces a different tree every time — just like real trees grown from the same seed in different spots.

This mimics "noise" in nature:
wind · sunlight · genetics · soil

Task 11
Maximum realism — the final tree
Realism enhancements added
def realtree(n, length, angle, scale): if n > 0: # taper: thicker trunk, thin twigs pen.pensize(max(1, n * 1.2)) # colour: brown trunk → green tips if n <= 2: pen.color("forest green") elif n <= 4: pen.color("dark green") else: pen.color("sienna") # random variation as before... jitter = random.randint(-20, 20) pen.forward(length) # ... (recursive calls same as Task 10)
Tapered thickness

Branch thickness is set to n × 1.2 pixels — so the trunk is thick and twigs are hair-thin. Real trees are structurally stronger at the base.

Colour transition

High-n levels (trunk, main boughs) are drawn in sienna. Mid-level branches shift to dark green. The outermost twigs (n ≤ 2) become forest green — simulating leaves.

Increased angle randomness

Jitter widened to ±20° at the outermost levels gives the canopy a more organic, windswept shape.

The big idea

Complex shapes found in nature — trees, coastlines, snowflakes — can be described by very simple recursive rules. One function with a base case and a recursive call is all it takes to grow an infinitely detailed fractal from nothing.

Technical challenge

Fixing the _tkinter library error on Mac required configuring the correct Python framework build — a reminder that environment setup is part of real programming.

Math encountered

Recurrence relations, geometric scaling (powers of the scale factor), and the connection between Fibonacci numbers and natural growth all surfaced naturally through the code.
